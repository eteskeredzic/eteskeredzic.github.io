C/C++ Optimizations

  1. Always avoid jumps and branches;

    • If possible, use switch/case whenever applicable. The compiler will (sometimes) optimize it to a lookup table. If you are forced to use if/else statements, put the most common ones first;
    • When calling a function, two jumps are performed;
    • Iteration is always better than recursion;
    • Never write short functions, either write the code directly, or use inline functions;
    • Avoid calling functions inside loops. Instead, call loops inside of functions.

  2. Consider how arrays are stored in memory;

    • N-dimensional (N > 1) arrays are always stored as 1D arrays in memory;
    • When accessing data, try to do it sequentially (simulate the way it is stored in memory);
    • Data is always fetched into cache in blocks. That means that, if you accessed arr[i][j], then arr[i][j+1] is probably already cached - you do not have the same guarantee for arr[i+1][j].

  3. Avoid local variables;

    • If your function only has a few local variables, the compiler can optimize those to be stored on the stack.

  4. Use shifting instead of multiplication and division;

    • Shifting left by N bits gives the same result as multiplying by 2^N;
    • Shifting right by N bits gives the same result as dividing by 2^N.

  5. Consider using table lookups if doing GPU or FPGA Programming;

  6. Be mindful of the operators you use;

    • For simple data types, always use +, -, *, / instead of +=, -=, *=, and /=;
    • For objects, the oposite is true.

  7. Whenever you can, use prefix operators (++x and --x) instead of postfix operators (x++ and x--);

  8. Try to simplify your math;

    • For example, if you have to divide by x many times, consider calculating y = 1/x first, and then doing multiplications by y instead of divisions by x;
    • Avoid mathematical functions if you don't need them. Checking if a > b gives the same resul as checking if sqrt(a) > sqrt(b) - for most use cases, at least;

  9. Some stuff specific to code:

    • If you're working in C++, consider running std::ios::sync_with_stdio(false); - this makes the cin and cout operations preform much faster;
    • Most compilers will optimize zero assingment (i.e. x = 0) to self-XORing (i.e. x ^= x). However, some won't, so considering doing the XORing yourself;
    • If the direction of a loop does not matter, consider replacing for(i = 0; i < n; ++i) with for(i = n; --i);
    • Printf and scanf are faster than cout and cin.
